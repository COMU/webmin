Resposta del servidor SMTP server en cas de rebuig per nom de host invàlid
invalid_hostname_reject_code
Aquest paràmetre especifica el codi de resposta del servidor SMTP quan un client viola la restricció anti-UCE reject_invalid_hostname que és:
Sintaxi incorrecta del nom de host HELO client (no compleix amb RFC)

Tingues en compte que el valor per defecte de postfix utilitzat per invalid_hostname_reject_code és el codi 501, i aquest codi a l'estàndard RFC 822 voldria dir 'Error de sintaxi als paràmetres o arguments'. Així, amb Postfix, un codi d'estat de 501 no reflecteix exactament l'explicació RFC822.

Per evitar la confusió, pots canviar-ho, per exemple, pel 574 (una altra raó és que el codi 501 és també el valor per defecte de postfix per altres menes d'errors), altrament no ho canviïs si no tens un complet coneixement de RFC 822.
