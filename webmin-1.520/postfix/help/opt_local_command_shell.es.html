Shell a usar para entrega a comando externo
local_command_shell
Este parámetro controla qué shell será usado para entrega a comando externo. Por defecto, los comandos externos son ejecutados diréctamente; los comandos son dados a /bin/sh sólo cuando contienen meta-caracteres de shell o comandos internos de shell.

Especifica un shell diferente cuando necesites más control sobre lo que los programas pueden ejecutar p.ej. archivos .forward.

"el shell restringido de sendmail" smrsh es el que la mayoría usará (smrsh es parte de recientes distribuciones de sendmail)

Nota: cuan se especifica un shell, es invocado incluso cuando el comando no contiene comandos internos de shell o meta-caracteres.

Ejemplo: /alguno/donde/smrsh -c
