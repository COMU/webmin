Puertos y Trabajo en Red
Puerto Proxy: Por defecto es 3128. Este valor dice en qué puerto escuchará tu servidor Squid por requerimientos http. Los navegadores cliente necesitarán ser configurados para conectarse al proxy en este puerto o la transparencia deberá también de ser configurada.

Puerto ICP: El número de puerto donde Squid envía y recibe requerimientos ICP hacia y desde cachés vecinas. Por defecto es 3130. Para desactivarlo usa el "0". Puede ser ignorado mediante la línea de comando -u.

Direcciones TCP de Entrada: Por defecto ninguna. Define el socket HTTP que acepta conexiones desde clientes y otras cachés.

Direcciones TCP de Salida: Por defecto es ninguna. Define el socket HTTP para conexiones realizadas a servidores remotos y a otras cachés.

Direcciones UDP de Salida: Por defecto es ninguna. Define el socket ICP que recibe paquetes de otras cachés.

Direcciones UDP de Entrada: Por defecto es ninguna. Define el socket para los paquetes ICP que se envían a otras cachés.

Búfer de recepción TCP: Medida del búfer TCP. Por defecto es el búfer de recepción del SO.
