Introducción

Este módulo le permite configurar el servidor DNS BIND (Berkeley Internet Name Daemon) en esta máquina. El servidor DNS es responsable de convertir nombres de máquina tales como www.foo.com en direcciones IP como 10.1.2.3, y direcciones IP en nombres de máquina.

Conceptos Básicos de DNS

Cada servidor DNS gestiona cierto número de dominios directos e inversos. Un dominio directo es algo como foo.com, y un dominio inverso es como 204.165.66. Para cada dominio el servidor DNS puede ser primario (el origen primario de los datos del dominio) o secundario (un respaldo de los datos del dominio).

Dominios Directos
Si su servidor DNS gestiona un dominio directo como foo.com, entonces será capaz de responder a preguntas sobre nombres de máquina dentro de este dominio. Para dominios para los que este servidor sea promario puede usar este módulo para añadir máquinas (y otros registros) al dominio. Para dominios secudarios puede especificar el(los) servidor(es) primario(s) desde los que descargar la información de las máquinas.

Dominios Inversos
Los dominios inversos mapean direcciones IP (como 203.165.66.77) a nombres de máquina (como www.foo.com). Un dominio inverso proporcinoa nombres a todas las direcciones IP de una red (como 203.165.66). Para dominios para los que este servidor sea primario este módulo le permite añadir registros de IP a nombre de máquina para el dominio. Para dominios secundarios puede especificar el(los) servidor(es) DNS desde los que descargarse los mapeos de direcciones IP.

Configurando el Servidor DNS

Si su sistema ya tiene un servidor DNS BIND en ejecución, entonces puede ignorar esta sección. Sin embargo, si Webmin detecta que el fichero de configuración del servidor DNS primario (habitualmente /etc/named.boot) no existe en su sistema, asumirá que aún no ha configurado el servidor DNS.

Para un uso normal en Internet, un servidor DNS debe disponer de una lista de servidores de nombres raíz que puedan responder a consultas externas a sus dominios. Si su servidor no está configurado, Webmin podrá :

  1. Descargar esta lista desde el servidor FTP de Internic en rs.internic.net. Suponiendo que la máquina donde ejecuta Webmin tiene conexión a Internet y que no está bloqueado por un cortafuegos, esta es la mejor opción.

  2. Usar la copia de la lista distribuída con Webmin. Puesto que los servidores de nombres raíz no cambian a menudo esto debería funcionar borrectamente.

  3. Configurar el servidor de nombres sólo para uso interno. Si está en una red sin conexión a Internet, esta es la opción que necesita.

Creando un Dominio Primario

Para crear un nuevo dominio primario, rellene el formulario bajo Añadir nuevo dominio primario en la página Servidor DNS. El significado de los campos es :
  • Mapeo
    Determina si éste es un dominio directo o inverso

  • Dominio/Red
    Para dominios directos, el dominio que añadir (como foo.com). Para dominios inversos, la red IP que añadir (como 203.123.221).

  • Nombre de fichero
    El nombre del fichero en que guardar la información del dominio. Si deja este campo en blando se generará automáticamente un nombre de ficheros adecuado.

  • email de contacto
    Cada dominio DNS debe tener asociado una dirección de correo electrónico, que debe ser el de la persona responsable del dominio.

  • Servidor de nombres
    Debe contener el nombre plenamente cualificado del servidor de nombres responsable de este dominio. Casi siempre será el mismo nombre que el de la máquina donde está ejecutando Webmin.
