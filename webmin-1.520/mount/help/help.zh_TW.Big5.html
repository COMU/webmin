前言

這個模組的作用在於簡單的掛載本地端磁碟或遠端檔案分享與虛擬記憶體. 在 Unix 系統能存取磁碟, 光碟, Jaz 磁片, 軟碟或檔案伺服器上的檔案之前, 它必須先被掛載載檔案系統的某個掛載點上. 系統同時也會維持一個所知道檔案系統的列表. 檔案系統也可以是暫時的, 在重新開機或卸載後就忘記它的存在.

所有的 Unix 系統在安裝完畢後至少都會在開機的過程中掛載一個檔案檔案系統, 而且在多數的狀況下都會有更多的. 永遠至少有一個被掛載在根目錄 / 的檔案系統, 而其他的可能是 /usr, /proc 或 /net. 除非您的 Unix 系統是無磁碟的工作站, 根目錄系統都是掛載自本地端的磁碟. 其他的檔案系統則可能是來自檔案伺服器, 本地端磁碟或甚至是神奇的檔案系統例如 /proc 並不包含任何真實的檔案.

一個特殊的狀況是虛擬記憶體. 您的系統永遠應該至少有一個虛擬記憶體交換檔案, 這被系統當作延伸的實體記憶體使用. 一個交換檔案可以是一個一般的檔案在某個檔案系統上, 或是本地端磁碟的其中一個分割區.

這個模組是經由讀取並更新檔案 /etc/fstab 或 /etc/vfstab 檔案以管理已知的檔案系統. 目前掛載的檔案系統可以經由讀取檔案 /etc/mtab 或 /etc/mnttab 所確認. Webmin 將會完全忽視任何已知檔案系統表個中的註解.

列出目前的檔案系統

檔案系統管理者的主要畫面是列出所有已知的檔案系統. 對於每一個檔案系統, 下列的詳細資料將會被顯示:
  • 掛載點
    這是檔案系統被掛載的目錄, 或是虛擬記憶體如果這是一個交換檔案. 每個掛載點上只能掛載一個檔案系統.

  • 檔案系統類別
    檔案系統類別的完整名稱. 依據掛載的方式 (本地端的或遠端的) 與磁碟格式化的方式顯示. 從本地端磁碟掛載的類別可能是 Linux 原生檔案系統 (Linux Native Filesystem), MS-DOS 檔案系統 或 Solaris Unix 檔案系統等等. 遠端掛載的則永遠是網路檔案系統.

    如果您想看到較短的檔案系統名稱 (例如 ext2fs 而不是 Linux 原生檔案系統), 請到模組組態改變是否使用長檔案系統類別的設定.

  • 位置
    掛載來源的設備檔案或遠端檔案系統名稱與目錄. 在 Unix 下面, 所有本地端的磁碟都有一個設備控制檔案在 /dev/ 之中, 並且可以用於存取實際的磁碟. 設備檔案與路徑像是 /dev/dsk/c0t1d0s3 或 /dev/hda1. 當您編輯或建立一個掛載時, Webmin 將會自動的轉換成對應的設備編號.

  • 是否使用中?
    如果檔案系統已經被掛載時為"是".

  • 是否為永久的?
    如過這個檔案系統是永久性的紀錄為"是". 對於"否"的檔案系統在卸載後便會被遺忘.

增加新的檔案系統

要增加一個新的掛載, 在列表下方選擇適當的檔案系統類別. 這會顯示一個表單讓您輸入掛載點位置, 掛載來源與其他必要的選項. 每一個檔案系統類別在不同的 Unix 版本間都會有不同的選項組, 而且每個類別都會有許多不同的可能掛載來源.

不論您選了哪一種檔案系統類別, 在建立掛載表單中都會有一些相同的欄位. 最重要的是"掛載點", 這是檔案系統將要被掛載的位置. 除了虛擬記憶體之外的每一中檔案系統都需要這個參數. 如果您輸入的目錄並不存在, Webmin 將會嘗試為您建立.

對於大多數的檔案系統您有 3 個選想可以用於儲存掛載:

  • 開機時儲存並掛載
    這個掛載將會有永久性的紀錄, 而且在系統開機時自動掛載之.

  • 儲存
    這個掛載將會有永久性的紀錄, 但是載系統開機時不會自動掛載之.

  • 不要儲存
    這個掛載將僅是暫時性的掛載.

對於大多數的檔案系統類別, 您可以選擇是否要立刻掛載. 同時選擇"不要掛載"與"不要儲存"是沒有有意義的, 而且會產生錯誤訊息.

然而有些檔案系統類別不會出現上面的選項, 例如:

  • Linux 下的虛擬記憶體
    這個類別是已經掛載並儲存的, 或卸載並刪除的. 儲存虛擬記憶體永遠是在開機時掛載的.

  • Linux 下的視窗網路
    這個檔案系統只能用 smbmount 命令掛載, 所以無法儲存.

  • 自動掛載器檔案系統
    永久的自動掛載器檔案系統永遠是在開機時掛載的.

所有的檔案系統都需要您選擇一個掛載來源與掛載選項. 因為來源與選項的特徵是依據作業系統與檔案系統的不同而變化的, 所以在下面分別說明:

  • Redhat Linux 4.0+ and Slackware Linux 3.0+
    對於大多數的 Linux 檔案系統類別而言, 掛載來源有下面幾種:
    • IDE 設備
      對於這個選擇, 您必須輸入 IDE 設備的字母與分割區. 設備字母對於 1st 磁碟通常是 a, 2nd 磁碟通常是 b, 而光碟機是 c. 而分割區則是您想要掛載的位置. 您可以使用 fdisk 指令以檢視磁碟機上的分割區.

    • SCSI 設備
      對於這個選項, 您必須輸入 SCSI 設備字母與分割區. 但設備字母不是 SCSI 的目標號碼 - 每個 SCSI 設備是依據作業系統找到的順序而給予子母.

    • 軟碟機
      這個選項只需要您輸入軟碟機設備編號. 對於第一個 (a 碟) 為 0, 第二個 (b 碟) 為 1.

    • 其他設備
      如果您選了這個選項, 您可以輸入任何您想掛載設備的路徑名稱. 對於要掛載磁碟陣列設備或是 PCMCIA 卡等 Webmin 不知道的設備時, 這是有用的.

    在一個本地端的 IDE 或 SCSI 磁碟分割區可以被掛載之前, 這個分割區必須已經被建立並且已經完成格式化. 這個分割區管理員無法用來做這件事. 您可以使用 Linux 系統所提供的 fdisk 與 mkfs 命令完成之.

    網路檔案系統 (NFS), 視窗網路檔案系統 (smbfs), 自動掛載器檔案系統 (auto) 與 虛擬記憶體 (swap) 的掛載類別使用不同的掛載來源. 請參照下面關於檔案系統的章節取得細節.

編輯一個現存的檔案系統

要編輯一個現存的檔案系統, 按下主畫面列表中的掛載點. 這會顯示與用於建立新掛載時相似的表單. 您可以如同建立一個新的掛載一樣的變更表單中的任何東西. (描述在前面).

一個現存的掛載可能是永久儲存的, 目前被掛載的或二者都是. 如果您改變成不儲存或掛載, 它將會從掛載列表中被刪除. 注意有些檔案系統類別沒有這個選項.

在 Unix 之中, 一個正被使用的目錄無法被卸載, 不論是程序或是使用者目前的路徑在其中子目錄之中. 相似的, 如果現存的記憶體空間小於交換但, 這個交換檔案無法被卸載. 當您改變了目錄, 來源或任何的選項之後, Webmin 會卸載並重新掛載這個目錄以套用變更. 當目錄或交換檔正被使用中而無法卸載時, 這將會失敗而且有錯誤訊息產生.
