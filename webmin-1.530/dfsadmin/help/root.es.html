Acceso de Root
Controla qué máquinas serán tratadas como si el usuario que entra fuera root. Si un cliente no está en la lista, incluso si puede montar la compartición como de sólo-lectura o de lectura-escritura, el servidor NFS tratará al usuario root del cliente como si fuera un Usuario Anónimo.
Las máquinas puede ser especificadas de la siguiente manera:

Nombre de máquina o dirección IP (como ftp.foo.com o 1.2.3.4)
Que coincida la máquina del mismo nombre o dirección
Grupo de Red (como ingeniería)
Que coincida cualquier máquina que sea miembro del grupo de red
Dominio DNS (como .foo.com)
Que coincida cualquier máquina del dominio
Red (como @10.254.1)
Que coincida cualquier máquina de la red
Red/máscara de red (como @10.254.1/24)
Que coincida cualquier máquina de la red

Además, cualquiera de las anotaciones de arriba puede llevar el prefijo -, queriendo esto decir que la máquina, grupo de red, dominio o red no tiene permisos para montar esta compartición. Esto puede ser útil si desea permitir el acceso a un conjunto de máquinas pero denegárselo a un miembro.
