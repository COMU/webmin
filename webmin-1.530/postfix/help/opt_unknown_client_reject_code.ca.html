Resposta del servidor SMTP en cas de rebuig per client desconegut
unknown_client_reject_code
Aquest paràmetre especifica el codi de resposta del servidor SMTP quan un client sense mapejat adreça-a-nom viola la restricció reject_unknown_clients, això és: Accés denegat: Nom de host client SMTP desconegut (sense registre DNS A o MX).

Tingues en compte que el valor per defecte utilitzat per postfix per a unknown_client_reject_code és el codi 450, i aquest codi en l'estàndard RFC 822 voldria dir 'Acció de correu demanada no executada: bústia ocupada o accés denegat' i demanaria el remitent de provar-ho un altre cop més tard (amb l'esperança que el DNS respondrà correctament més tard).
No ho canviïs si no tens un coneixement complet de RFC 822.
