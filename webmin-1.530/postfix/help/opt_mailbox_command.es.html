Comando externo a usar en lugar de la entrega a buzón
mailbox_command
Este parámetro especifica el comando externo opcional a usar en lugar de la entrega a buzón. El comando es ejecutado como el destinatario con los valores de entorno HOME, SHELL y LOGNAME adecuádamente puestos. Excepción: la entrega a root es hecha como el $default_user.

Otras variables de entorno de interés: USER (nombre de usuario destinatario), EXTENSION (extensión de dirección), DOMAIN (parte de dominio de la dirección) y LOCAL (la parte local de la dirección).

Al contrario que otros parámetros de configuración de Postfix, este parámetro no está sujeto a sustitución de parámetro. Esto es para hacer más fácil el especificar sintáxis de shell (mira el ejemplo debajo).

Evita meta-caracteres de shell ya que fuerzan a Postfix a ejecutar un caro proceso de shell. Procmail en sí mismo es ya bastante caro.

SI USAS ESTO PARA ENTREGAR CORREO A LO LARGO DEL SISTEMA, DEBES CONFIGURAR UN ALIAS QUE ENVÍE CORREO PARA ROOT A UN USUARIO REAL.

Ejemplos:

  • /alguno/donde/procmail
  • /alguno/donde/procmail -a "$EXTENSION"
