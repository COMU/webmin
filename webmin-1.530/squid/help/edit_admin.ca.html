Opcions Administratives
Executa com a usuari Unix: Per defecte, és 'nobody' o 'squid', depenent del sistema. Si el cau s'executa com a root, canviara el seu ID efectiu/real a aquest valor.

Executa com a grup Unix: Per defecte, és 'nogroup' o 'squid', depenent del sistema. Si el cau s'executa com a root, canviara el seu ID de grup efectiu/real a aquest valor.

Reialme d'autenticació de proxy: Especifica el nom de reialme que s'utilitza per informar el client per a l'autenticació de proxy (part del text que l'usuari veurà quan se li demani l'usuari i la contrasenya).

Adreça de correu de l'administrador del cau: Per defecte, és 'webmaster' o 'root', depenent del sistema. L'adreça de correu de l'administrador local que rebrà correu si es mor el cau.

Nom de host visible: Si vols presentar un nom de host especial en missatges d'error, etc., llavors defineix això. Altrament, s'utilitzarà el valor de retorn de gethostname(). Si tens múltiples caus en un clúster i tens errors amb el redireccionat IP, has de posar que tinguin noms individuals.

Nom de host únic: Si vols tenir múltiples màquines amb el mateix nom de host visible, llavors has de donar a cada màquina un nom de host únic per tal que es puguin detectar els redireccionaments.

Host d'anunci de cau: Nom del host on s'enviaran els missatges d'anunci. Busca més informació sobre les jerarquies dels caus a The Squid FAQ. El defecte és 'tracker.ircache.net', quan s'ha activat l'anunci amb un cert període.

Port d'anunci de cau: Port del host on s'enviaran els missatges d'anunci. Per defecte, és el 3131 quan l'anunci està activat.

Període d'anunci: Freqüència dels anuncis. Activa els anuncis a una jerarquia de caus establint-ho a '1 dia'. Per defecte, és '0', que desactiva l'anunci.
